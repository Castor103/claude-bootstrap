# AI 코딩 웹 서비스 보안 점검 — 팀 공유용 요약

**작성일:** 2026.06.16
**대상:** BE+FE 개발팀 (AI 활용 개발, 외부 노출 서비스)
**목적:** 보안 가이드 작성 전 팀 내 공통 인식 맞추기

---

## 왜 이게 필요한가

- AI 생성 코드의 **24.7%에 보안 결함** 존재 (인간 코드 대비 약 2배)
- AI 활용 개발자의 70%가 취약점 발견 경험, 그 중 1/5이 심각한 사고(데이터 유출, 서비스 장애)로 이어짐
- AI가 추가한 종속성의 49%에 알려진 취약점 포함
- **실제 사고:** Lovable(170개 앱 사용자 데이터 공개 노출), Moltbook(150만 인증 토큰 유출), EnrichLead(클라이언트 사이드만 보안 → 브라우저에서 유료 기능 우회)

---

## AI가 특히 자주 실수하는 패턴 5가지

1. **시크릿 하드코딩** — API 키, DB 비밀번호를 소스코드에 직접 삽입. Git 히스토리에 영구 잔류
2. **인증 없는 API 엔드포인트** — REST API를 만들면서 인증 체크를 빠뜨림. URL만 알면 누구나 접근 가능
3. **SQL 인젝션** — 문자열 연결로 SQL 생성 (`f"SELECT ... WHERE id = '{input}'"`)
4. **XSS** — 사용자 입력을 이스케이프 없이 HTML에 삽입 (`dangerouslySetInnerHTML`)
5. **클라이언트 사이드만의 보안** — 프론트엔드에서만 권한 체크하고 서버측 검증 누락. 브라우저 개발자 도구로 즉시 우회

---

## 체크리스트: 배포 전 반드시 확인할 것

### 데이터 & 입력

- [ ] 모든 사용자 입력은 **백엔드에서 검증** (프론트 검증은 UX일 뿐, 보안이 아님)
- [ ] SQL은 **매개변수화 쿼리** 사용 (`db.execute("...WHERE id = ?", (input,))`)
- [ ] 사용자 콘텐츠 출력 시 **HTML 이스케이프** 적용
- [ ] 파일 업로드: 허용 타입 제한 + 크기 제한 + 앱 외부 저장
- [ ] 사용자 입력에 `../` 같은 경로 조작이 들어올 수 있음 — 경로 화이트리스트 적용

### 인증 & 세션

- [ ] 인증은 **검증된 라이브러리** 사용 (직접 구현 금지)
- [ ] 비밀번호: **bcrypt 또는 argon2로 해시+솔트**. 평문/MD5/SHA-1 저장 절대 금지
- [ ] 쿠키 보안 플래그 3종: **HttpOnly**(JS 접근 차단), **Secure**(HTTPS만), **SameSite**(CSRF 방지)
- [ ] **인증 ≠ 인가**: "로그인 했다" ≠ "이 데이터에 접근할 수 있다". 모든 API에서 역할/권한 검증
- [ ] 관리자 계정에는 최소한 **MFA(다단계 인증)** 적용

### 인프라 & 설정

- [ ] **HTTPS 강제화** — HTTP → HTTPS 자동 리다이렉트, SSL 인증서 자동 갱신
- [ ] 시크릿(API 키, DB 암호, JWT 시크릿)은 **환경변수 또는 시크릿 매니저**로 관리
- [ ] `.env`는 `.gitignore`에 등록. Git 히스토리에 한 번이라도 커밋되면 영구 잔류
- [ ] CORS: **와일드카드(`*`) 금지**. 프론트엔드 도메인만 명시적으로 허용
- [ ] `npm audit` / `pip-audit` **정기 실행**. 사용하지 않는 패키지 제거
- [ ] AI가 추천한 패키지가 실제로 존재하는지 확인 (패키지 할루시네이션)

### 앱 동작

- [ ] **Rate Limiting** 적용 — 로그인 브루트포스, API 남용, DoS 방지
- [ ] 에러 메시지: 사용자에게는 일반적 메시지만. **스택 트레이스, DB 스키마 노출 금지**
- [ ] 로그에 비밀번호, 토큰, 개인정보 기록 금지

## Claude로 보안 점검하는 방법 (우리 팀 기준)

- **`/cso`** — OWASP Top 10 + 시크릿 스캔 + 종속성 분석 종합 감사
- **`/review`** — PR diff 기준 보안 체크
- **프롬프트 활용** — "보안 엔지니어 역할로 이 코드를 검토해줘"
- **가드레일 파일** — `.claude/rules/security.md`에 금지 패턴(하드코딩 시크릿, eval(), 와일드카드 CORS 등)을 등록하면 AI가 생성 시점에 알아서 피함

## 최소한 지금 당장 할 수 있는 3가지

1. **gitleaks pre-commit 훅 설치** — 시크릿이 커밋되는 것을 사전 차단
2. **`.claude/rules/security.md` 작성** — AI 가드레일 (위 금지 패턴 등록)
3. **배포 전 `/cso` 또는 `/review` 실행** — 보안 감사를 습관화

## 핵심 원칙 (팀 논의용)

1. **AI 코드는 초안이다** — 검토 안 된 서드파티 코드와 동일하게 취급
2. **인프라 우선** — 코드 결함이 있더라도 HTTPS, 방화벽, CORS로 침해 확산을 막는 구조
3. **Shift Left** — 배포 후가 아니라 커밋 전에 잡는다
4. **AI가 설명 못하면 미구현** — "이 보안 조치를 어떻게 구현했어?" 물었을 때 답 못하면 없는 거임
5. **자동화** — 사람 기억에 의존하지 말고 CI 파이프라인에 보안 게이트 설치
